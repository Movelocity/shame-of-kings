## 2. 原则（继承，不改）

来自 `archives/proposal.md` §2 / §5.4，仍有效：

1. 渲染 / 逻辑分离；飘字走 Three.js Sprite，不走 React。
2. 固定步长 `1/60`；技能逻辑纯 TS 可单测。
3. 数据驱动英雄（JSON + loader）；木人桩永远静止。
4. **M4 前不引入新依赖**；不大改 `src/engine/` 核心接口（除非为碰撞/重置所必需）。
5. **M4 前不写元歌 / 镜 / `map.yaml` / 寻路 / 视野 / Minimap 代码。**

### 2.1 本版新增原则

| 原则 | 含义 |
|---|---|
| **机制先于皮肤** | 亚瑟 JSON 里已有字段必须有 runtime；没有 runtime 的字段不算「成型」 |
| **练习场先于大地图** | 障碍碰撞、重置、横屏 DoD 优先于 `coords` / 塔类型扩展 |
| **闸口够用即停** | G1/G2 已存在；不为「将来并行」再加抽象 |
| **文档跟代码走** | README 进度表与本文件同步；过时表不准留 |

## 8.1 桌面端操作（练习场）

| 按键 | 作用 |
|---|---|
| **WASD / 方向键** | 移动 |
| **鼠标左键** | 点击地面寻路 |
| **J** | 普攻（默认锁敌） |
| **K** | 普攻 · 优先小兵（P2 单位类型落地后生效；当前练习场等同 J） |
| **L** | 普攻 · 优先防御塔（同上） |
| **U / I / O / P** | 技能 1 / 2 / 3 / 4（对应 hero kit 槽位 `1`–`4`） |

移动端仍使用右下角 SkillHud 触控；槽位显示为数字 `0`–`3`，无键盘热键。

映射实现见 `src/engine/input/desktop-skill-hotkeys.ts`；内部 hero kit 槽位约定不变（`0` 普攻 + `1`–`3` 主动）。
